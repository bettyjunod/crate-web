Масштабируемость. Простота. Быстрота.

Crate – это неразделяемое, документо-ориентированное кластерное хранилище данных с возможностью поиска: быстрый и мощный массивно-масштабируемый сервер для приложений с обработкой больших объемов данных, Интернета вещей или анализа в режиме реального времени. Это приложение со 100% открытым исходным кодом.

Crate.IO является хранилищем данных, обладающей преимуществом документо-ориентированной базы данных NoSQL, мощным поиском (Elasticsearch включен в качестве библиотеки), использованием SQL.

Мощный и сверхбыстрый

Crate – это единый мощный сервер для табличных данных, полуструктурированных записей и двоичных объектов (BLOBs). Приложение является высоко доступным и предлагает массово-параллельный доступ в реальном времени, позволяя отвечать на миллиарды записей меньше чем за секунду.

Нулевое администрирование

Все узлы Crate.IO являются равными. Добавление узлов в стандартное оборудование или облако выполняется также просто, как ввод одной командной строки в терминале. Кластер Crate имеет функции самонастройки и самовосстановления.

Почему Crate?

Crate.IO – это масштабируемое решение в реальном времени, которое предлагает большие преимущества стоимости и эффективности. Crate заняло уникальное место, объединив SQL (например, MySQL), NoSQL (например, MongoDB) и поиск (Lucene/ES).

БЫСТРО И ЛЕГКО ИСПОЛЬЗОВАТЬ

Использует SQL для чтения и записи ответов в течение менее одной секунды.

ВЫСОКАЯ ДОСТУПНОСТЬ

Кластеры Crate являются надежными и устойчивыми. Неисправные узлы удаляются, новые узлы добавляются.

CRATE ПО СРАВНЕНИЮ С ДРУГИМИ ТЕХНОЛОГИЯМИ

Приложение Crate.IO было создано, чтобы разработчики интенсивных серверов не нуждались в объединении нескольких технологий для хранения документов, двоичных объектов и поиска в реальном времени. Мы также хотели помочь разработчикам избежать ручной работы, связанной с настройкой, фрагментацией, репликацией и другими операциями, необходимыми для содержания большого хранилища денормализованных данных в хорошем состоянии. Мы хотели создать простое, отказоустойчивое и массивно-масштабируемое хранилище данных, которое можно использовать на одном компьютере, нескольких компьютерах или в облаке.
Хранилище данных для приложений с обработкой больших объемов данных должно быть надежным, горизонтально масштабируемым и поддерживать следующие пять требований:
(1) хранение документов, (2) двоичных объектов, (3) выполнение повторного поиска (4) выполнение запросов (аналитика) и (5) прямые изменения структуры данных при необходимости.

Возможно, разработчики могут просто объединить несколько технологий и получить такие же результаты? Вот некоторые примеры:

  • MongoDB + ElasticSearch + GridFS. GridFS поддерживает хранение двоичных объектов, ElasticSearch – поиск, MongoDB будет делать все остальное. Хотя трудно изменить структуру данных, масштабировать и управлять многоузловой средой.
  • Riak + Solr + Rados. Rados для хранения двоичных объектов, Solr для поиска и Riak для остального. Очень похоже на первый вариант и имеет такие же проблемы.
  • CouchDb (хранение документов и аналитика) + ElasticSearch (поиск) + HDFS (хранение двоичных объектов, архивация) + Hadoop (аналитика и изменения). Это решение имеет множество проблем, связанных с Hadoop, а также стабильностью при большом объеме чтения и записи.

Давайте посмотрим, как Crate справляется с этими 5 требованиями при одновременной поддержке самовосстановления и самонастройки хранилища данных.

1. Хранение документов
Crate.IO сохраняет документы с помощью встроенной технологии Lucene и ElasticSearch. Приложение Crate добавило функции дампа/восстановления для всех крупных обновлений версий, что переводит «восстановление на определенный момент времени» на новый уровень.
2. Хранение двоичных объектов
Хранение двоичных объектов выполняется прозрачно с помощью реализации двоичных объектов Crate, встроенных в семантику кластера Elasticsearch для распространения, репликации и размещения. Оно просто сохраняет двоичные объекты прямо в файловой системе пользователя.
3. Поиск документов
Crate использует Elasticsearch для поиска – это лучшее из доступных скоростных средств.
4. Анализ данных в режиме реального времен с помощью SQL
Crate позволяет анализировать данные в режиме реального времен с помощью SQL через анализатор, планировщик и механизм выполнения распределенных запросов. Это добавляет расширенные функции группирования и сортировки для обеспечения функциональности SQL типа “group by … having ..” и “order by”. Представьте себе MapReduce, но в реальном времени.
5. Выполнение изменений без общего переделывания
Помните старые добрые времена, когда реструктуризацию всей реляционной базы данных можно было выполнить с помощью одного сценария SQL? Crate реализует эту функцию с помощью записи результата запроса прямо в один или несколько индексов. Это требование стало более необходимым в базах данных документов, где данные часто хранятся в денормализованном формате.

Клиенты и инструменты

Crate поставляется со встроенным интерфейсом администрирования. Интерфейс командной строки (оболочка Crate – CraSh) позволяет выполнять интерактивные запросы. Клиент Python Crate является самым продвинутым и интегрируется с SQLAlchemy.
