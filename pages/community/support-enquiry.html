Reporting Issues

{{ description }}

If you have general questions about Crate, e.g. how to use certain Crate specific data types or how to perform fulltext search, etc. just go ahead and drop us an email at support@crate.io, join our Crate.IO Slack Community or chat using the Intercom widget on our website.

If you experience any issues with a Crate cluster you're already running you can submit a report that will help us better understand and solve your problem.

Because in the end we're happy when our users are happy!

Ideally your report would include the following points (if applicable):

  • Crate logs with the relevant lines
  • Stack trace from server or client
  • System/cluster information gathered using Crash

The first two points are self explanatory, but the third point can be difficult, especially when you don't know what you're looking for. So we've built a tool which finds what you don't know you're looking for. All you need to do is copy/paste the output into the email.

Here's how it works:

Provide System/Cluster Information

You can generate the report by simply running crash with the --sysinfo argument:

$ crash --sysinfo --hosts example.com:4200

--------------------------------------------------------------------------------
...
For debugging purposes you can send above listed information to support@crate.io

You can either copy the output into an email or pipe it into a file and attach the file to your email:

$ crash --sysinfo --hosts example.com:4200 > report.txt

Alternatively, while using Crash simply type \sysinfo and hit enter:

$ crash --hosts example.com:4200
cr> \sysinfo

Requirements

  • Crate version >= 0.54.0
  • Crash >= 0.15.x

Additional Information

Usually we also ask for additional information, such as:

  • Disk type (HDD, SSD, etc.)
  • Network interface
  • Cloud environment (AWS, Azure, GCE, etc.)

This helps us to deal with your issues faster and more effectively.

If you have a lower version of Crash (< 0.15) or Crate (< 0.54) you can also try to run our external sysinfo-queries which is hosted on Github.
